Transforming Skewed Continuous Features¶

A dataset may sometimes contain at least one feature whose values tend to lie near a single number, but will also have a non-trivial number of vastly larger or smaller values than that single number. Algorithms can be sensitive to such distributions of values and can underperform if the range is not properly normalized. With the census dataset two features fit this description: 'capital-gain' and 'capital-loss'.

Run the code cell below to plot a histogram of these two features. Note the range of the values present and how they are distributed.

For highly-skewed feature distributions such as 'capital-gain' and 'capital-loss', it is common practice to apply a logarithmic transformation on the data so that the very large and very small values do not negatively affect the performance of a learning algorithm. Using a logarithmic transformation significantly reduces the range of values caused by outliers. Care must be taken when applying this transformation however: The logarithm of 0 is undefined, so we must translate the values by a small amount above 0 to apply the the logarithm successfully.

Run the code cell below to perform a transformation on the data and visualize the results. Again, note the range of values and how they are distributed.

Metrics and the Naive Predictor¶

CharityML, equipped with their research, knows individuals that make more than \$50,000 are most likely to donate to their charity. Because of this, *CharityML* is particularly interested in predicting who makes more than \$50,000 accurately. It would seem that using accuracy as a metric for evaluating a particular model's performace would be appropriate. Additionally, identifying someone that does not make more than \$50,000 as someone who does would be detrimental to *CharityML*, since they are looking to find individuals willing to donate. Therefore, a model's ability to precisely predict those that make more than \$50,000 is more important than the model's ability to recall those individuals. We can use F-beta score as a metric that considers both precision and recall:

$$ F_{\beta} = (1 + \beta^2) \cdot \frac{precision \cdot recall}{\left( \beta^2 \cdot precision \right) + recall} $$

In particular, when $\beta = 0.5$, more emphasis is placed on precision. This is called the F$_{0.5}$ score (or F-score for simplicity).

Looking at the distribution of classes (those who make at most \$50,000, and those who make more), it's clear most individuals do not make more than \$50,000. This can greatly affect accuracy, since we could simply say "this person does not make more than \$50,000" and generally be right, without ever looking at the data! Making such a statement would be called naive, since we have not considered any information to substantiate the claim. It is always important to consider the naive prediction for your data, to help establish a benchmark for whether a model is performing well. That been said, using that prediction would be pointless: If we predicted all people made less than \$50,000, CharityML would identify no one as donors.

Random Forest:¶

Random forest usually works better on most problems just outside the box. Ensembles works better, if the models are different from each other (learn different things) and it pays off to combine their predictions to compute the final one.

RF use bootstrapping technique / bagging to train multiple independent tress and then combine the results from those to compute the final results. Bagging is implemented using the sampling with replacement technique, whereby each tree is given a different dataset to train on, and thus making them different from each other. Further to reduce the variance, randomization is incorporated in each of the decision trees. During the train, the algorithm randomly chooses subset of the features to decide what feature to split on. This makes them more stable and less prone to outliers fitting.

To combine the results, there are multiple techniques. For classification problems, majority voting or avg probabilities are used. For regression problems, mean/average is performed on all the predictions to get the final results.

RF / Decision Forests have been used in previous versions of Microsoft Kinect to detect the human body pose (https://www.microsoft.com/en-us/research/publication/real-time-human-pose-recognition-in-parts-from-a-single-depth-image/?from=http%3A%2F%2Fresearch.microsoft.com%2Fpubs%2F145347%2Fbodypartrecognition.pdf, http://www.i-programmer.info/news/105-artificial-intelligence/2176-kinects-ai-breakthrough-explained.html) among many other applications.

Another advantage of the RF algorithm is that, they are intrinsically designed for parallel execution / training, since all the decision trees are independent of each other. This works great with all the distributed parallel machine learning frameworks where multiple trees in the RF are trained separately on part of the hardware and therefore making them scalable.

Weakness(es) - Due to randomization, we usually train RF with many trees to get more stable /accurate results. This makes RF trained model very complex and is slow at making run time predictions. Another problem with RF (regressor) is its lack of ability to extrapolate to unseen outcome. This may become the problem with regression tasks where you usually see an increasing / decreasing trend which is outside the space of what was available at the time of training.

Gradient Boosting:¶

Gradient boosting is another ensemble technique where we train multiple models and combine them to get final predictions. However, they are different from RF in multiple ways. GBMs use boosting as the technique to train the complete model.

The idea here is that, we start with weak learners and iteratively combine them to get final predictions. We start with the first weak learner, find out the samples where it is making errors, and boost those samples (increase the sample weights) so that they could be better dealt with by another weak learner in the next iterations. With this approach, every additinal weak learned in the sub-sequent iterations improves upon the already added weak learners by focusing on the samples where error rate is high. Another way to put that would be, each weak learner exploits the underlying weight distribution to find the optimal weak learned at every iteration. This is different from RF, where the distribution usually remains uniform.

The strength of gradient boosting is to reduce the loss/error marginally with every sub-sequent iteration.

Weakness(es) - Serieal training for all the iterations (since next iteration boosts the previous iteration errors). Easier to overfit if the weak learners become strong learners.

Gradient boosted techniques, with the especial implementations like XGBoost known to be performing better than most other algorithms and have been used widely to win various kaggle competitions in the past. In addition, gradient boosting technique has been leveraged in many fields - Travel time prediction (http://www.sciencedirect.com/science/article/pii/S0968090X15000741), Predicting causes of network congestion (http://ieeexplore.ieee.org/abstract/document/7161501/?reload=true), and for predictive analytics on sensor data (http://ieeexplore.ieee.org/abstract/document/7037726/) etc.

from sklearn.metrics import fbeta_score, accuracy_score

def train_predict(learner, sample_size, X_train, y_train, X_test, y_test): 
    '''
    inputs:
       - learner: the learning algorithm to be trained and predicted on
       - sample_size: the size of samples (number) to be drawn from training set
       - X_train: features training set
       - y_train: income training set
       - X_test: features testing set
       - y_test: income testing set
    '''
    
    results = {}
    
    # TODO: Fit the learner to the training data using slicing with 'sample_size'
    start = time() # Get start time
    learner = learner.fit(X_train[:sample_size], y_train[:sample_size])
    end = time() # Get end time
    
    # TODO: Calculate the training time
    results['train_time'] = end - start

    # TODO: Get the predictions on the test set,
    #       then get predictions on the first 300 training samples
    start = time() # Get start time
    predictions_test = learner.predict(X_test)
    predictions_train = learner.predict(X_train[:300])
    end = time() # Get end time
    
    # TODO: Calculate the total prediction time
    results['pred_time'] = end - start
    
    # TODO: Compute accuracy on the first 300 training samples
    results['acc_train'] = accuracy_score(y_train[:300], predictions_train)
        
    # TODO: Compute accuracy on test set
    results['acc_test'] = accuracy_score(y_test, predictions_test)
    
    # TODO: Compute F-score on the the first 300 training samples
    results['f_train'] = fbeta_score(y_train[:300], predictions_train, 0.5)
        
    # TODO: Compute F-score on the test set
    results['f_test'] = fbeta_score(y_test, predictions_test, 0.5)
       
    # Success
    print "{} trained on {} samples. Training time {} seconds, and Prediction time {} seconds.".format(learner.__class__.__name__, sample_size, results['train_time'], results['train_time'])
        
    # Return the results
    return results
