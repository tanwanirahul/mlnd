Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 3 - Training and Testing¶

  • What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?

Hint: Think about how overfitting or underfitting is contingent upon how splits on data is done.

Answer: Purpose of learning algorithms is not to memorize the training data but to find the generic pattern so as to predict for unforeseen cases. One way we can test the validity of the model in this regard is by actually keeping aside some data and treating them as unforeseen cases (not yet seen cases). The best way we can achieve this is by spliting our dataset into training and testing. We learn from training and evaluate the model on testing. Since the purpose of the model is to predict for future cases, it is always a best practice to report the performance on the test set as the peformance of the model.

When we see a very low error (high accuracy) on training set and very high error (low accuracy) on test set, that is an indicator that the model is not well generalized and is a situation of overfitting. On the other hand, if the model is biased or doesn't react / learn from siginificant changes in the independent variables, we consider this to be the condition of underfitting.

Because we evaluate the model based on test set, it is very important to make sure that the test set is representative of the actual dataset / future cases. It is almost always a best practice to suffle the data to avoid any intrinsic ordering before spliting it or to do random sampling (this is what train_test_split does).

Answer:

  • There is alwasy a trade off between maximizing training and test data. We need max training set to learn all the pattern and nuances in the data, whilst more the testing data, better the evaluation and confidence on the evaluation score. We address this, by creating multiple copies of training and test datasets with K-Fold cross validation technique. The idea is, to divide the data into K-chunks (partitions/folds), use K-1 for training and 1 for testing. We repeat this train and test set preparation for K times. Each time we consider a different fold among K-folds for testing. This way, every sample in the dataset is atleast once being used in the test set.

  • With grid space search, K-Fold helps to address the issue to relying on one test set to choose the winner among all the models. If the test set is not representative, or is biased towards a particular combination of those parameters, the grid search technique may falsely choose the less optimum model based on the evaluation score. With K-Fold, each combination of parameters are trained and evaluated K times on different folds of training and test set and hance reduces the biases/un-representativeness of test set for evaluation.

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {"max_depth": range(1, 11)}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor, params, scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
